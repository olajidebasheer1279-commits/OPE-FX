OPE-FX has reached a stable state and is now ready for production deployment.

DO NOT modify any existing business logic, calculations, trading features, UI, database schema, authentication, analytics, Cloudinary integration, notifications, reviews, journal, or settings.

The objective is ONLY to convert the project from a Replit Runtime application into a standard production-ready application.

==========================
PRODUCTION MIGRATION
==========================

1. Remove every Replit-specific dependency and runtime requirement.

This includes:
- @replit/vite-plugin-runtime-error-modal
- @replit/vite-plugin-cartographer
- @replit/vite-plugin-dev-banner
- any Replit-only runtime helpers

2. Update artifacts/ope-fx/vite.config.ts

Remove the requirement for:

- BASE_PATH
- REPL_ID

The frontend must build correctly using standard Vite defaults.

Use:

base: "/"

unless another production-safe base path is required.

The app must no longer depend on Replit runtime variables.

==========================
EXPRESS PRODUCTION
==========================

Configure the Express backend so that production serves the Vite build.

Requirements:

When NODE_ENV=production

Serve:

artifacts/ope-fx/dist

using Express static middleware.

Unknown frontend routes must return index.html.

API routes must continue functioning exactly as before.

==========================
BUILD PIPELINE
==========================

Create a production build pipeline.

Running

pnpm run build

must:

1. Build frontend
2. Build backend
3. Produce a complete deployable application

No manual steps.
