# OPE-FX

A professional-grade Forex trading journal and analytics dashboard for traders of Forex, Metals, Indices, and Synthetic Indices.

## Architecture

**Monorepo** managed by pnpm workspaces with two artifact services and shared libraries:

| Path | Role |
|---|---|
| `artifacts/ope-fx/` | React 19 + Vite 7 frontend (Tailwind CSS v4, shadcn/ui, TanStack Query v5) |
| `artifacts/api-server/` | Express 5 API server with Drizzle ORM, Clerk auth, live market data, alert engine |
| `lib/db/` | Drizzle schema + database client (PostgreSQL) |
| `lib/calc-engine/` | Zero-dependency PnL/pip calculation engine |
| `lib/api-client-react/` | Type-safe React hooks generated by Orval |

## Running the project

All three workflows start automatically:

- **Frontend** (`artifacts/ope-fx: web`) — Vite dev server
- **API Server** (`artifacts/api-server: API Server`) — builds then starts with esbuild
- **Mockup Sandbox** (`artifacts/mockup-sandbox: Component Preview Server`) — component preview server

## Environment variables & secrets

| Key | Notes |
|---|---|
| `DATABASE_URL` | Runtime-managed — automatically provided by Replit |
| `SESSION_SECRET` | Secret — already set |
| `CLERK_SECRET_KEY` | Secret — already set; from your Clerk dashboard → API Keys |
| `CLERK_PUBLISHABLE_KEY` / `VITE_CLERK_PUBLISHABLE_KEY` | Set in shared env — `pk_test_*` key from Clerk |
| `CLOUDINARY_CLOUD_NAME` / `CLOUDINARY_API_KEY` | Set in shared env — for trade screenshot uploads |
| `CLOUDINARY_API_SECRET` | Secret — needed for Cloudinary uploads if used |
| `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` / `VAPID_SUBJECT` | Optional — for web push notifications |

## Database migrations

Run schema migrations against Replit's built-in PostgreSQL:

```bash
pnpm --filter @workspace/db run push
```
